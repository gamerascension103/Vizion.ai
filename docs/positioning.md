# Vizion Corp — Positioning

The product, the mission, the trust model, and the language Vizion uses to describe itself. Source of truth for site copy, brief copy, and any external writing. Anything that conflicts with this doc is wrong by default; push back in chat before shipping.

---

## What Vizion sells

**A dashboard that makes a small business's money leaks visible to its decision-maker.**

That is the v1 product. It is industry-specific — the metrics, the analyses, the visualizations are tuned to the operator's industry. It is sold as the ground-zero engagement and it is what every new client gets first.

Everything else is expansion.

---

## The package ladder

The site, the sales motion, and the long-term roadmap all map to three stages. Each stage is a real product that stands on its own. Each is a natural upgrade from the last.

### Package 1 — The Dashboard (ground zero, what gets sold today)

Industry-specific dashboard that surfaces the money leaks the operator can't see right now. This is what Vizion sells, ships, and bills for today. Every engagement starts here.

### Package 2 — The Custom Agent (next step)

A custom AI agent, trained on the operator's industry, role, and tools, that watches the metrics the dashboard surfaces and helps the operator decide faster. Operator approves every action. Adds on top of the dashboard; doesn't replace it.

### Package 3 — Handler Agents (long-term vision)

Multi-agent system installed inside the business. One handler agent per department head, each trained to that role's industry, tasks, and tools. An internal debrief rhythm runs across them. Operator approves every action. This is the end-state Vizion is building toward.

The site leads with Package 1. Package 2 and 3 are shown as the second and third acts — the credible expansion path, not the lead pitch.

## The trust model

This is the philosophical core. It is not a feature.

**Operator approves every action.** Always. The same trust model Jake, the chat, and Cascade run on internally. The agent proposes, the human decides, the agent executes, the system debriefs.

The wider AI agent market is selling **replacement of decisions.** Vizion sells **acceleration of decisions with the human still in the chair.** That distinction is real, and it deserves real estate on the site.

The proposal → approval → execution → debrief loop is literally a state machine. It is depictable. It should be visible somewhere on the site.

## Who it's for

Small business decision-makers. Owner-operators or small-team execs who can feel money slipping but cannot see exactly where.

Not enterprise. Not Fortune 500. Not "mid-market" in the consultancy sense. The customer is the person who would actually open a dashboard at 7am and use it.

## In-house vocabulary

Words used freely in copy:

- **Money leaks** — what the product finds. The noun.
- **Visibility / making the invisible visible** — the core verb.
- **The dashboard** — the v1 deliverable. Not "AI platform," not "analytics solution," not "intelligence layer."
- **Decision latency** — the cost being reduced. Specific. Operator-resonant.
- **Debrief** — the rhythm being installed. Borrowed from aviation, military, sports. Implies discipline and repetition.
- **Handler agents** — the v3 architecture. One per department head.
- **Operator** / **operator approves** — the human in the chair. Used as a literal phrase, not paraphrased.
- **Industry-specific** — used freely. "Vertical-specific" or "tailored" are weaker substitutes.

Words to avoid because the entire competitive field uses them:

> transform, unlock, scale, operationalize, harness, leverage, journey, empower, AI-powered, intelligent, hybrid intelligence, responsible AI, beyond pilots, at scale

If a draft contains any of those, rewrite it.

## Implications for site structure

The site leads with the dashboard. The agentic vision is the second act, framed as expansion, not lead.

Recommended top-to-bottom shape:

1. **Hero** — the dashboard, or rather the *experience* of the dashboard: money leaks becoming visible. CSS/SVG animation territory. The animation is a preview of the actual deliverable, not decoration.
2. **What it sees** — how the system surfaces leaks. Industry-specific examples as proof, not as a menu.
3. **How the agent works** — proposal → approval → execution → debrief, made visible. The trust model gets real estate here. This is where Package 2 enters the conversation.
4. **Where it's going** — handler agents, the long-term vision. Package 3. Framed as the path, not the pitch.
5. **How an engagement starts** — concrete first step. No vague "book a discovery call."

This structure works for v1 (one industry shown in proof) and v2+ (more industries added) without rebuilding the site.

---

## Industry strategy

Industries are surfaced as **proof**, not as the front door. The site does not need to choose a single hero industry at v1 — it needs one credible industry example per major proof section. Additional industries get added to those sections over time as the case studies accumulate.

If a single hero industry emerges as the dominant client base in the next few months, the hero copy can be tuned to it without rebuilding the rest of the site.
